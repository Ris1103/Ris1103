# 👋 Hi, I'm Rishank Gautam (@Ris1103)

### AI Engineer · Building Production LLM Systems & Intelligent Agents

I'm an AI Engineer focused on building production-grade AI systems — from LLM-powered microservices and multi-agent pipelines to RAG systems and real-time inference APIs. I specialize in closing the gap between research-grade models and reliable, scalable products that actually ship.

My 2 years of hands-on experience span the full AI product lifecycle: data pipelines, fine-tuning, agentic architectures, and robust backend systems. I care about latency, cost, observability, and making AI systems that developers and end-users can trust.

### 🚀 Focus Areas

- **LLM Engineering:** Prompt engineering, structured outputs, tool use, context window management, and model evaluation
- **Agentic AI:** Designing multi-agent systems, orchestration frameworks (LangGraph, CrewAI), and autonomous pipelines
- **RAG & Retrieval Systems:** Hybrid search, re-ranking, chunking strategies, vector store optimization
- **LLM Ops & Observability:** Tracing, evaluation, cost monitoring, and deployment best practices
- **AI Microservices:** Production FastAPI services with async I/O, caching, rate limiting, and OpenAI/Azure OpenAI integration
- **Data Engineering:** CDR/telecom data, SQL optimization, ETL pipelines, and multi-sheet Excel reporting

### 🧠 What I Know Cold

**LLM Engineering**
- Prompt engineering — few-shot, chain-of-thought, ReAct, self-consistency
- Structured output extraction (function calling, JSON mode, instructor)
- Context window optimization, chunking, and token budget management
- Fine-tuning with LoRA/QLoRA and instruction tuning (SFT, DPO, RLHF concepts)
- LLM evaluation: RAGAS, DeepEval, LLM-as-judge patterns

**Agentic Systems**
- Multi-agent orchestration with LangGraph and CrewAI
- Tool/function calling and MCP (Model Context Protocol)
- Memory systems: episodic, semantic, and working memory in agents
- Human-in-the-loop (HITL) workflows
- Streaming and async agent execution

**RAG Pipelines**
- Hybrid search: dense + sparse (BM25) with RRF re-ranking
- Document parsing, chunking strategies (semantic, recursive, late chunking)
- Metadata filtering and query routing
- Evaluation: context precision, recall, faithfulness, answer relevancy

**Production AI**
- Async FastAPI services with background task queues (Celery + Redis)
- LLM gateway patterns: caching, fallback routing, cost control
- Tracing and observability with Langfuse / LangSmith / Arize
- Docker-based deployment on Azure
